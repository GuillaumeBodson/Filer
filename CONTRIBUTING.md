# Contributing

The Git workflow, branching model, commit conventions, PR process, CI, and
branch-protection setup are documented canonically in
[`project documents/11-git-workflow.md`](project documents/11-git-workflow.md).

In short: branch off `main` (`feat/…`, `fix/…`), use
[Conventional Commits](https://www.conventionalcommits.org), open a PR (even
solo), let CI go green, and **squash-merge**. Never commit secrets.
